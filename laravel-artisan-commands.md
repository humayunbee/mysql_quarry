## ১. প্রজেক্ট রান এবং হেল্প কমান্ড
প্রজেক্ট চালু করা বা কমান্ডের লিস্ট দেখার জন্য এগুলো ব্যবহার করা হয়।

| কমান্ড | কাজ / বিবরণ |
| :--- | :--- |
| `php artisan serve` | লোকাল কম্পিউটারে লারাভেল ডেভেলপমেন্ট সার্ভার চালু করার জন্য। (ডিফল্ট ইউআরএল: `http://127.0.0.1:8000`) |
| `php artisan list` | লারাভেলে এভেইলেবল বা উপলব্ধ সবগুলা কমান্ডের একটা কমপ্লিট লিস্ট দেখার জন্য। |
| `php artisan help [command_name]` | নির্দিষ্ট কোনো কমান্ড কীভাবে ব্যবহার করতে হয় এবং সেটার কাজ কি, তা বিস্তারিত জানার জন্য। *যেমন: `php artisan help make:model`* |

## ২. Make কমান্ড (নতুন ফাইল বা কম্পোনেন্ট তৈরি)
নিজে থেকে ম্যানুয়ালি কোড না লিখে এই কমান্ডগুলোর মাধ্যমে খুব দ্রুত আর্কিটেকচারাল ফাইল তৈরি করে নেওয়া যায়।

| কমান্ড | কাজ / বিবরণ |
| :--- | :--- |
| `php artisan make:controller ControllerName` | নতুন একটি কন্ট্রোলার (Controller) ফাইল তৈরি করার জন্য। |
| `php artisan make:model ModelName` | ডাটাবেজ টেবিলের সাথে যোগাযোগ করার জন্য নতুন একটি মডেল (Model) তৈরি করতে। |
| `php artisan make:migration create_tables_name` | ডাটাবেজে নতুন টেবিল তৈরি বা পরিবর্তন করার জন্য মাইগ্রেশন ফাইল তৈরি করতে। |
| `php artisan make:seeder SeederName` | ডাটাবেজে টেস্ট বা ডামি ডাটা ইনসার্ট করার জন্য সিডার ফাইল তৈরি করতে। |
| `php artisan make:factory FactoryName` | একসাথে অনেকগুলো ডামি ডাটা জেনারেট করার নিয়ম বা ফ্যাক্টরি তৈরি করতে। |
| `php artisan make:middleware MiddlewareName` | এইচটিটিপি রিকোয়েস্ট ফিল্টার করার জন্য মিডলওয়্যার (Middleware) তৈরি করতে। |
| `php artisan make:mail MailName` | ইমেইল পাঠানোর জন্য নতুন মেইল ক্লাস তৈরি করতে। |

> 💡 **প্রো-টিপ (All-in-One):** আপনি চাইলে এক কমান্ডেই মডেল, ডাটাবেজ মাইগ্রেশন এবং রিসোর্স কন্ট্রোলার তৈরি করতে পারেন:
> ```bash
> php artisan make:model ModelName -mcr
> ```
> *এখানে `-m` = Migration, `-c` = Controller, এবং `-r` = Resource Methods.*

## ৩. Migration কমান্ড (ডাটাবেজ ম্যানেজমেন্ট)
ডাটাবেজের স্কিমা তৈরি, পরিবর্তন বা রিলেট করার জন্য এই কমান্ডগুলো ব্যবহৃত হয়।

| কমান্ড | কাজ / বিবরণ |
| :--- | :--- |
| `php artisan migrate` | প্রজেক্টের সব পেন্ডিং মাইগ্রেশন ফাইল রান করে ডাটাবেজে নতুন টেবিলগুলো তৈরি করে। |
| `php artisan migrate:rollback` | ঠিক শেষবার যে মাইগ্রেশন ব্যাচটি রান করেছিলেন, সেটাকে রিভার্ট বা আনডু (Undo) করে। |
| `php artisan migrate:reset` | ডাটাবেজের সব টেবিল রোলব্যাক (মুছে) করে দেয়। |
| `php artisan migrate:refresh` | ডাটাবেজের সব টেবিল প্রথমে রোলব্যাক করে এবং তারপর আবার শুরু থেকে নতুন করে মাইগ্রেশন রান করে। |
| `php artisan migrate:fresh` | কোনো রোলব্যাক মেথড না ডেকে সরাসরি ডাটাবেজের সব টেবিল ড্রপ (Drop) করে দেয় এবং একদম শুরু থেকে মাইগ্রেশন রান করে। |
| `php artisan db:seed` | আপনার তৈরি করা সিডার ফাইলগুলো রান করে ডাটাবেজে ডামি ডাটা পুশ করার জন্য। |

## ৪. Cache এবং Optimization কমান্ড
প্রজেক্টের পারফরম্যান্স বাড়ানো বা কোনো কনফিগারেশন পরিবর্তন করার পর যদি তা ব্রাউজারে রিফ্লেক্ট না করে, তখন এগুলো ব্যবহার করা হয়।

| কমান্ড | কাজ / বিবরণ |
| :--- | :--- |
| `php artisan config:cache` | `.env` এবং সব কনফিগারেশন ফাইলগুলোকে একসাথে কম্বাইন করে ক্যাশ ফাইল তৈরি করে (স্পিড বুস্টের জন্য)। |
| `php artisan config:clear` | কনফিগারেশন ক্যাশ ফাইলটি মুছে ফেলে (নতুন `.env` ভেরিয়েবল এড করলে এটি দিতে হয়)। |
| `php artisan route:cache` | রুটগুলো দ্রুত লোড করার জন্য রুট ক্যাশ ফাইল তৈরি করে। |
| `php artisan route:clear` | রুটের ক্যাশ ক্লিয়ার করার জন্য। |
| `php artisan view:clear` | কম্পাইলড হওয়া ব্লেড টেমপ্লেটের (Blade Views) ক্যাশ ফাইলগুলো পরিষ্কার করার জন্য। |
| `php artisan optimize:clear` | **[সবচেয়ে দরকারী]** এক কমান্ডেই ক্যাশ, রুট, কনফিগারেশন, ভিউ—সবকিছুর ক্যাশ একসাথে পরিষ্কার (Clear) করার জন্য। |

## ৫. অন্যান্য দরকারি কমান্ড

| কমান্ড | কাজ / বিবরণ |
| :--- | :--- |
| `php artisan tinker` | একটি ইন্টারেক্টিভ REPL টার্মিনাল চালু করে, যার মাধ্যমে সরাসরি কমান্ড লাইনেই পিএইচপি কোড এবং ডাটাবেজ মডেলের কাজ লাইভ টেস্ট করা যায়। |
| `php artisan route:list` | প্রজেক্টে এই মুহূর্তে যতগুলো ইউআরএল বা রুট (Routes) ডিফাইন করা আছে, তাদের মেথড ও কন্ট্রোলারের নামসহ একটি সুন্দর টেবিল দেখায়। |
| `php artisan storage:link` | লারাভেলের সুরক্ষিত `storage/app/public` ফোল্ডারটিকে সরাসরি `public/storage` ফোল্ডারের সাথে সিমলিংক (Symlink) করে দেয়, যাতে আপলোড করা ফাইল ব্রাউজারে দেখা যায়। |
| `php artisan key:generate` | প্রজেক্টের সিকিউরিটির জন্য `.env` ফাইলে একটি শক্তিশালী `APP_KEY` জেনারেট করে বসিয়ে দেয় (নতুন প্রজেক্ট সেটআপের সময় বাধ্যতামূলক)। |
| `php artisan down` | প্রজেক্টটিকে লাইভ সার্ভারে সাময়িকভাবে বন্ধ (Maintenance Mode) করার জন্য। |
| `php artisan up` | মেইনটেন্যান্স মোড বন্ধ করে প্রজেক্টটিকে আবার সবার জন্য লাইভ করার জন্য। |
